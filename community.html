COMMUNITY

Contribute with scientific context intact.

The most useful contributions improve accessibility, reproducibility, data provenance and visual legibility. New survey layers should retain their original selection and uncertainty context.

Start here

  • Read the project README and data policy.
  • Open an issue for a visual, data or documentation proposal.
  • Use supplied templates for source-layer suggestions.
  • Keep raw survey archives and multi-million-row tile stores outside normal Git history.

Useful contribution areas

  • Mobile and keyboard navigation.
  • Accessible explorer states and reduced-motion support.
  • Validated source adapters with column-level checks.
  • Documentation, tutorial notebooks and source provenance.

Scientific guardrails

Survey footprints and source uncertainties should remain visible in the interface. New layers should preserve the original catalogue context and cite their data release.

Read CONTRIBUTING.md, DATA_POLICY.md and COMMUNITY.md in the repository.
